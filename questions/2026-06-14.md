## Q1. 实现 `DeepReadonly<T>`

**题目**

将一个对象类型（包括嵌套对象）的所有属性递归地设为只读。

```ts
type DeepReadonly<T> = /* 你的实现 */

type Config = {
  server: { host: string; port: number };
  flags: { debug: boolean };
};

type ReadonlyConfig = DeepReadonly<Config>;
// server.host 不可修改，server.port 不可修改，以此类推
```

**答案**

```ts
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K];
};
```

**解说**

- 映射类型 `[K in keyof T]` 遍历所有属性，加 `readonly` 修饰符。
- 条件类型判断属性值是否为 `object`：是则递归，否则直接保留原类型。
- 注意 `Function`、`Date`、`Array` 都是 `object`，如需排除可在条件中细化。

**应用场景**

Redux / Zustand store 的 state 类型定义；防止在 reducer 外部意外修改状态。

## Q2. 实现 `Flatten<T>`

**题目**

将嵌套数组类型拍平为一维数组元素类型。

```ts
type Flatten<T> = /* 你的实现 */

type A = Flatten<number[]>;           // number
type B = Flatten<Array<Array<string>>>;  // string
type C = Flatten<string>;             // string（非数组原样返回）
```

**答案**

```ts
type Flatten<T> = T extends Array<infer Item> ? Flatten<Item> : T;
```

**解说**

- `infer Item` 从数组类型中提取元素类型。
- 递归调用直到 `T` 不再是数组为止。
- 这是条件类型 + `infer` 的经典组合。

**应用场景**

处理多维数组 API 返回值的类型推断；编写通用工具函数时约束参数类型。

## Q3. 函数重载：`parseInput`

**题目**

实现一个函数 `parseInput`，满足以下签名：
- 传入 `string` → 返回 `number`
- 传入 `number` → 返回 `string`
- 传入 `boolean` → 返回 `boolean`

```ts
const a = parseInput("42");   // 类型：number
const b = parseInput(42);     // 类型：string
const c = parseInput(true);   // 类型：boolean
```

**答案**

```ts
function parseInput(x: string): number;
function parseInput(x: number): string;
function parseInput(x: boolean): boolean;
function parseInput(x: string | number | boolean): number | string | boolean {
  if (typeof x === "string") return Number(x);
  if (typeof x === "number") return String(x);
  return x;
}
```

**解说**

- TypeScript 函数重载分两部分：若干个**重载签名**（无函数体）+ 一个**实现签名**（有函数体）。
- 实现签名对外不可见，调用时只能匹配重载签名。
- 实现体内部需自行做类型收窄（`typeof` / `instanceof`）。

**应用场景**

SDK 设计中同一函数根据参数类型返回不同结构；避免滥用 `any` 的工厂函数。

## Q4. 条件类型 `IsNever<T>`

**题目**

实现类型工具 `IsNever<T>`，判断 `T` 是否为 `never`，返回 `true` 或 `false`。

```ts
type A = IsNever<never>;   // true
type B = IsNever<string>;  // false
type C = IsNever<0>;       // false
```

**答案**

```ts
type IsNever<T> = [T] extends [never] ? true : false;
```

**解说**

- 直接写 `T extends never ? true : false` 会因为**分布式条件类型**导致结果也是 `never`（当 `T = never` 时分发无元素）。
- 用 `[T] extends [never]` 将 `T` 包裹进元组，**禁用分发**，才能正确判断。

**应用场景**

类型测试工具（`type-challenges`）；条件分支中排除 `never` 类型的分支。

## Q5. 提取函数参数类型 `Parameters<T>`（手动实现）

**题目**

不使用内置 `Parameters`，手动实现同等效果。

```ts
type MyParameters<T extends (...args: any) => any> = /* 你的实现 */

function foo(a: number, b: string): boolean { return true; }
type P = MyParameters<typeof foo>; // [a: number, b: string]
```

**答案**

```ts
type MyParameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;
```

**解说**

- `infer P` 放在参数位置，TypeScript 会将完整的参数元组类型绑定到 `P`。
- 函数约束 `T extends (...args: any) => any` 确保只对函数类型使用此工具。
- 同理，`ReturnType<T>` 将 `infer` 放在返回值位置。

**应用场景**

HOC / 中间件包装时，自动提取原函数签名并透传参数类型。

## Q6. 模板字面量类型：`EventMap`

**题目**

给定事件名数组，自动生成 `on<EventName>` 风格的 handler 映射类型。

```ts
type Events = "click" | "focus" | "blur";

type EventHandlerMap = /* 你的实现 */
// 期望结果：
// {
//   onClick: () => void;
//   onFocus: () => void;
//   onBlur:  () => void;
// }
```

**答案**

```ts
type Capitalize<S extends string> =
  S extends `${infer F}${infer R}` ? `${Uppercase<F>}${R}` : S;

type EventHandlerMap<T extends string> = {
  [K in T as `on${Capitalize<K>}`]: () => void;
};

type Result = EventHandlerMap<Events>;
```

**解说**

- 映射类型的键重映射（`as` 子句）可以动态生成键名。
- 模板字面量类型 `` `on${Capitalize<K>}` `` 将 `"click"` 变为 `"onClick"`。
- TypeScript 4.1+ 内置 `Capitalize`、`Uppercase` 等字符串工具类型。

**应用场景**

React 组件 props 自动生成事件回调类型；事件总线类型安全封装。

## Q7. 协变与逆变：函数参数类型兼容性

**题目**

以下代码哪一行会报错？说明原因。

```ts
type Animal = { name: string };
type Dog = { name: string; breed: string };

let animalFn: (a: Animal) => void = (a) => console.log(a.name);
let dogFn:    (d: Dog)    => void = (d) => console.log(d.breed);

// (A)
animalFn = dogFn;

// (B)
dogFn = animalFn;
```

**答案**

**(A) 会报错**，(B) 合法。

```
// 报错：Type '(d: Dog) => void' is not assignable to type '(a: Animal) => void'
animalFn = dogFn; // ❌
dogFn = animalFn; // ✅
```

**解说**

- **函数参数类型是逆变的（contravariant）**：参数类型"更宽"的函数可以赋值给参数类型"更窄"的变量。
- `animalFn` 接受 `Animal`，但 `dogFn` 内部访问 `d.breed`——若调用者只传 `Animal`（没有 `breed`），会运行时出错，因此 TS 拒绝。
- `animalFn` 赋给 `dogFn` 安全：调用者会传 `Dog`（有 `name`），`animalFn` 只访问 `name`，没问题。

**应用场景**

理解回调函数类型兼容性、事件处理器赋值、HOF 类型推导。

## Q8. `infer` 提取 Promise 内层类型

**题目**

实现 `Awaited<T>`（不使用内置），支持任意嵌套深度。

```ts
type MyAwaited<T> = /* 你的实现 */

type A = MyAwaited<Promise<string>>;                    // string
type B = MyAwaited<Promise<Promise<number>>>;           // number
type C = MyAwaited<Promise<Promise<Promise<boolean>>>>; // boolean
```

**答案**

```ts
type MyAwaited<T> = T extends Promise<infer U> ? MyAwaited<U> : T;
```

**解说**

- 每一层递归用 `infer U` 拆开一层 `Promise`。
- 当 `T` 不再是 `Promise` 时，返回 `T` 本身。
- TypeScript 4.5+ 已内置 `Awaited<T>`，原理相同。

**应用场景**

async 函数返回值类型推断；将 Promise 链的最终类型传递给后续处理函数。

## Q9. 类型守卫 + 泛型：`assertDefined`

**题目**

实现断言函数 `assertDefined<T>`，若值为 `null | undefined` 则抛出错误，否则收窄类型为 `T`。

```ts
function assertDefined<T>(val: T | null | undefined, msg?: string): asserts val is T {
  /* 你的实现 */
}

let name: string | null = getName();
assertDefined(name, "name must be defined");
// 此行之后，name 的类型为 string
console.log(name.toUpperCase()); // 无 TS 错误
```

**答案**

```ts
function assertDefined<T>(
  val: T | null | undefined,
  msg = "Value is null or undefined"
): asserts val is T {
  if (val === null || val === undefined) {
    throw new Error(msg);
  }
}
```

**解说**

- `asserts val is T` 是**断言签名**（assertion signature），TypeScript 3.7 引入。
- 调用此函数后，TS 控制流分析认为 `val` 一定是 `T`（排除了 `null | undefined`）。
- 与类型谓词 `val is T` 不同：类型谓词用于返回 `boolean`，断言签名用于无返回（或抛错）的函数。

**应用场景**

运行时参数校验；在 Express 路由、测试断言中收窄可空类型。

## Q10. 递归类型：`DeepPartial<T>`

**题目**

实现 `DeepPartial<T>`，将对象所有属性（包括嵌套）变为可选，数组元素类型保持不变。

```ts
type DeepPartial<T> = /* 你的实现 */

type User = {
  id: number;
  profile: {
    name: string;
    address: { city: string; zip: string };
  };
  tags: string[];
};

type PartialUser = DeepPartial<User>;
// id?: number
// profile?: { name?: string; address?: { city?: string; zip?: string } }
// tags?: string[]   ← 数组元素不递归
```

**答案**

```ts
type DeepPartial<T> = T extends Array<infer U>
  ? Array<U>
  : T extends object
  ? { [K in keyof T]?: DeepPartial<T[K]> }
  : T;
```

**解说**

- 先判断是否为数组：是则保持 `Array<U>` 不递归（避免把 `string[]` 变成 `(string | undefined)[]`）。
- 再判断是否为对象：是则映射所有键并加 `?` 可选修饰，递归处理值类型。
- 否则原样返回（`string`、`number` 等基础类型）。

**应用场景**

PATCH 接口 body 类型定义；表单局部更新时的类型约束；测试 mock 数据构造。
